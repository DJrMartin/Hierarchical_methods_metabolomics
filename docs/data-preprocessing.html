An Example of Metabolomics Analysis

Chapter 2 Data preprocessing

This document is intended as a tutorial for analyzing metabolomics data from clinical or pre-clinical studies. It is structured into several steps.

The first step is to preprocess the metabolomics data to ensure it is suitable for downstream analyses.

2.1 Environment setup

To ensure reproducibility and avoid package conflicts, it is recommended to work in a clean R environment.

  1. Create a virtual environment (optional but recommended) with renv::init().

  2. Install the required packages listed in requirements.txt using the following command.

# Installer BiocManager
if (!requireNamespace("BiocManager", quietly = TRUE)) {
  install.packages("BiocManager")
}

# Lire requirements.txt
packages <- readLines("requirements.txt")

# Fonction simple de dispatch
install_if_needed <- function(pkg) {
  if (!requireNamespace(pkg, quietly = TRUE)) {
    tryCatch({
      install.packages(pkg)
    }, error = function(e) {
      BiocManager::install(pkg, ask = FALSE, update = FALSE)
    })
  }
}

# Installer tous les packages
invisible(lapply(packages, install_if_needed))

2.2 Importing data and metadata

The first operation is to REMOVE ADDUCTS from the matrix generated by MS-DIAL. This step currently needs to be performed manually. Then, I store all data and metadata into a single folder in my R project.

In most R projects, all data are stored in a dedicated folder (e.g., /data).

For convenience, I store both data and metadata into a single .rda file, which is structured as follows:

.rda
├── PHYSIO
│   ├── Groups of clinical variables
│   │
│   └── Clinical data
│       └── (table de données cliniques)
│
└── METABO
    ├── METABOLOMICS_PLASMA
    │   └── (données métabolomiques plasma)
    │
    ├── LIPIDOMIC_RBC
    │   └── (lipidomique globules rouges)
    │
    └── LIPIDOMIC_PLASMA
        └── (lipidomique plasma)

To access nested elements, use the $ operator.

Warning: Sample IDs may not be aligned across datasets. You must carefully match IDs before integrating data.

2.4 Mapping metabolites to KEGG pathways

Then, we have to map metabolites to KEGG pathways using this workflow:

  1. Extract KEGG IDs (column 5 of name_map.csv).

  2. Map KEGG IDs to pathways.

  3. Build pathway-specific metabolite matrices.

This is implemented in the function hierarchical_creation(), available in the C2VN R package.

source("fct/hierarchical_creation.R")

X.normalised = hierarchical_creation(X, # Rawdata of the metabolomics matrix
                           PATH_MetaboAnalyst = "data/name_map.csv", # path to the name_map.csv
                           cutoff = 5, # Number of minimum metabolites to create a pathway.
                           normalisation = TRUE) # Normalisation PQN: TRUE of FALSE

## We can also suppress non human-pathway.
X.normalised$X_list = X.normalised$X_list[-grep("plant", names(X.normalised$X_list))]

Output description

The function returns:

  1. A list of metabolite matrices per pathway.

  2. Associated physiological system labels.

  3. Metabolites not assigned to any pathway (i.e., haven’t got KEGG IDs) or insufficiently represented to construct a pathway.

To access nested elements, use the $ operator. For instance, we can get the names of the recognized pathways.

2.4.1 Limiting pseudo-replication (optional but recommended)

Some metabolites appear in many pathways ( > 5), which can lead to pseudo-replication issues in downstream analyses (the cutoff can be changed).

Here, we propose a simple strategy:

  1. Limit each metabolite to a maximum number of pathways, and

  2. Remove redundant occurrences when they do not add information

Redundancy in metabolomics is expected due to biological networks. This approach reduces statistical bias while preserving structure. The threshold (0.95) can be adjusted depending on the dataset. This step is optional but recommended for multivariate analyses, limiting the curse of dimensionnality.

2.4.3 Redundancy filtering strategy

The idea is to remove a metabolite from a pathway if its removal does not change the structure of the data.

We compare pathway structure using PCA-based distance matrices (i.e., Mantel test), and we keep at least 3 metabolites per pathways (can be changed).

X.preprocess <- X.normalised
suppressed = c()

for(i in a_traiter){
  for(j in 1:length(X.normalised$X_list)){
    
    df <- X.preprocess$X_list[[j]]
    if(nrow(df)>3){
      
      distance.matrix <- cor(as.numeric(dist(FactoMineR::PCA(t(df[which(rownames(df)!=i),]), graph = F, scale.unit = TRUE)$ind$coord)),
                           as.numeric(dist(FactoMineR::PCA(t(df), graph = F, scale.unit = TRUE)$ind$coord)))
      
      if(distance.matrix > 0.95 & distance.matrix < 1){
        
        X.preprocess$X_list[[j]] = df[which(rownames(df)!=i),]
        suppressed = c(suppressed, 
                       paste(rownames(df)[rownames(df)==i], "in the following pathways:", names(X.preprocess$X_list)[j]))
        
      }
    }
  }
}
